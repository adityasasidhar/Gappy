PHASE 0

Goal Stripping

Every AI is trained to validate user premises. PANCHAI intercepts incoming goals and splits the biased desired outcome from the raw physical constraints and facts.

Anti-Sycophancy Anchor

Agents literally cannot tell you what you want to hear because the stripping protocol obscures your preferred outcome from them.

INPUT/OUTPUT COMPARISON Goal: "Validate launching this app in 2 days."
Stripped: "Evaluate readiness. Specs: 12 features, 2 test files, 1 developer."
PHASE 1

Pre-Mortem Forcing

Before voting, agents operate in isolated passive mode. Each agent must submit a mandatory failure analysis questionnaire before they are allowed to see others or argue.

Destroys Anchoring Bias

Empirically proven to prevent groupthink. Agents must commit to writing what will fail before defending their reasoning bias.

MANDATORY AGENT TEMPLATE "If this fails, it will fail because: ______"
"The assumption most likely wrong is: ______"
"The stakeholder most harmed is: ______"
PHASE 2

Registry & Precedents

PANCHAI maps task semantics to a client's local agent capability registry, dynamically creating the council. It pulls relevant historical precedents from the client's Files silo.

Domain-Agnostic Engine

One codebase. The council composition and contextual memories change dynamically per client pod registry. Zero hardcoded agents.

LEMMA PRIMITIVE RAG method: "HYBRID" (Semantic + BM25 keyword)
retrieval: client_pod/memory/verdicts/*.md
PHASE 3

Active Debate Loop

A frontier model moderator (Brain) reads prior arguments, identifies the core axes of disagreement, and issues personalized challenge prompts. Cap at 3 rounds + 1 bonus round.

Deliberation Forcing

Agents cannot restate prior positions; they must defend their logic against conflicting logic, leading to compromise or reframing.

ACTIVE MODERATION PATTERN Brain reads -> Pinpoints disagreement -> Injects target prompt to Agent-N -> Loop resets.
PHASE 4

Verdict Synthesis

Once rounds lock, the Brain parses positions and counts votes. Standard tasks require Simple Majority (2/3); High-stakes require Supermajority (3/3) consensus.

Structured Decision output

Generates a locked verdict record, confidence score, minority dissent logs, and recommended enterprise actions.

VERDICT JSON FORMAT { "verdict": "ESCALATED", "confidence_score": 0.58, "hitl_required": true }
PHASE 5

HITL & Silo Archiving

An escalation matrix (ties, low confidence, high divergence, or critical pre-mortem) triggers a human review flow node. The final resolution is stored in client Files silo.

Epistemic Ceiling Safety

Prevents auto-approval timeouts and creates a federated memory audit trail for compliance.

ESCALATION TRIGGERS Confidence < 60% | Divergence == HIGH | Pre-mortem == CRITICAL | Vote == TIE

How PANCHAI Safeguards Enterprise AI

1. Blind Tribunal

By stripping preferred outcomes, agents evaluate facts in isolation. Groupthink and validation loops are structurally impossible.

2. Pre-Mortem Forcing

Commitment to failure modes prior to argument shifts LLM behavior from optimization validation to risk discovery.

3. Active Moderation

Personalized challenge prompts force agents to directly critique each other's assumptions instead of repeating scripts.

4. Siloed Archiving

Deliberations are stored as permanent markdown verdicts in independent client pods, creating transparent audit rails.
